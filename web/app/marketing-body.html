三步流程 · 模板 → AGENT → 微信

从一份模板，到活在微信里的 AI Agent。

平台在后台依次完成数据目录分配、配置生成、专属运行环境启动。整个过程通常在数十秒内完成。

STEP 01

选模板，起名字

从公开模板库里挑一个人格——私人助理、学习陪伴、记账伙伴。给它取个名字，提交。

tmpl = assistant-zh@v3
name = "我的私人助理"
✓ 已锁定模板版本
STEP 02

等待 Agent 进入 Running

平台异步供应：分配独立数据目录、复制 SOUL.md / USER.md / skills、启动专属运行环境。可以离开，回来再看。

› provision data dir · ok
› inject persona · ok
› status running
STEP 03

扫码，绑定微信

启动微信绑定 → 实时拿到二维码 → 微信里扫一扫 → 凭据加密落盘。从此发消息，它就在微信里回。

为什么选 AgentForge

把 Agent 工程化里那些恶心的东西，全部收敛进平台。

模板化创建

从预置模板出发，几次点击就能创建一个独立运行的 Agent。版本被锁定，已创建的 Agent 不会因模板更新而被改坏。

多 Agent 并行托管

每个 Agent 拥有独立的运行时和数据目录。工作助理、学习陪伴、生活记账互不干扰，端口、配置、日志都不会再互相打架。

微信扫码全流程

二维码生成、状态轮询、凭据加密落盘、网关启动、断线重连——全部在平台里完成。你只需要拿起手机扫一下。

数据持久化

Agent 的人格、记忆、对话历史、微信凭据全部保存在独立数据目录中。运行时容器销毁与重建，都不会让这些东西消失。

多用户隔离

每个用户只能访问自己的 Agent。任何人都无法读取、修改、操作他人的 Agent 数据或运行时——隔离是平台的硬约束。

异步供应 · 错误可恢复

从分配数据目录到运行时启动是一条可观测的事件流。Agent 启动失败、扫码超时都能进入可恢复状态，平台提供清晰的重试入口。

密钥仅在服务端

模型 API key、微信凭据、平台 token 只在服务端使用。前端响应、日志、错误信息里看不到任何敏感凭据。

运行历史与状态可见

查看 Agent 的运行状态、关键事件流、错误信息。它今天有没有掉线、为什么掉线，一眼就能看明白。

默认收敛的安全策略

仅扫码本人发起的私信被回复，群聊默认禁用。让 Agent 留在你的会话里，而不是被陌生人或群聊滥用。

典型场景 · 三类个人开发者

三种把 Agent 装进生活的方式。

场景 01 · 验证想法

给一个 Agent 创意一周不到的验证窗口。

独立开发者有一个 Agent 创意，但不想花一周时间研究框架、写运行时、接消息平台。用 AgentForge：选模板 → 命名 → 扫码。今晚就能在微信里跟它对话验证。

想法 → 模板 → Agent → 微信 · 一晚
不写代码 · 不配 YAML · 不开 VPS
场景 02 · 长期托管

会写 Agent 的极客，懒得维护服务器。

已经会写 Agent，但不想再自己维护服务器、容器、二维码登录态、断线重连。把这些交给平台：异步供应、独立运行时、断线自动恢复，加密凭据持久化。

专注 Agent 行为 而不是基础设施
升级运行时 → 数据目录不变 · 无需重新扫码
场景 03 · 自然交互

让 AI 进入你已经在用的微信会话。

不再开 App、网页、ChatGPT 标签页。Agent 就活在你的微信会话里——给它发消息像给朋友发消息，回复进入聊天列表，跟其他对话排在一起。它默认只回复你本人，群聊默认禁用。

微信私信 → 本人专属 · 群聊默认禁用
凭据加密 · 不出现在前端 / 日志

微信扫码 · 可视化演示

四步绑定，从二维码到能聊。

点击右侧任意一步，看 Agent 与微信连接的完整过程。整段流程在平台内完成；凭据加密落盘后，运行时重建也不会要求你重新扫码。

  • 1
    生成二维码 Agent 进入 Running 后，启动微信绑定，平台实时返回登录二维码与有效期。

    用微信扫一扫 状态轮询从「等待扫码」切换到「等待确认」，二维码上出现已扫描覆盖。

    开始接收微信消息 Agent 默认只回复扫码本人的私信。从此你给它发消息，它就在你熟悉的会话里回。
